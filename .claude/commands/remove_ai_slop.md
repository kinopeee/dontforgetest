# AI っぽいスラップ（不要生成物）除去

## 概要

このブランチの差分（`main` 比）から、AI 生成に起因する不自然/不要な要素だけを取り除き、コードベースの慣例に揃える。

## 対象（例）

- ファイル全体のスタイルと不一致な「説明過多」コメント（自明/重複/冗長）
- 過剰な防御的チェック、不要な `try/catch`（信頼されたコードパス、上位で握るべき例外など）
- 型問題回避だけの `any` キャスト、無意味な型アサーション
- 使われない変数/関数、不要な分岐、過度に一般化されたユーティリティ化

## 非対象（やらない）

- 仕様変更・挙動変更（リファクタは最小限）
- 「好み」の書き換え（別スタイルへの大改造、命名の総入れ替え等）

## 実行手順

1. `main...HEAD` の差分に限定して、上記「対象」を機械的に除去する（関係ない箇所は触らない）
2. 可能なら、削除/簡略化によって型・lint・テストが通ることを確認する

```bash
cd "$(git rev-parse --show-toplevel)" && npm run lint && npm run typecheck && npm test
```

## 最終報告（必須）

- 何を消した/簡略化したかを **1〜3 文** で要約（ファイル名 or 変更種類ベース）
